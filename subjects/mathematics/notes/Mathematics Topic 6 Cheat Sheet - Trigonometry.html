6.1–6.2 Right-angled triangles Core

📐 Pythagoras

6.1 · right angles only
c² = a² + b²
  • Hypotenuse c = √(a²+b²) — always the longest side, opposite the right angle.
  • A shorter side a = √(c²−b²).

Example: a ladder 5 m long rests 3 m from a wall → height = √(5²−3²) = √16 = 4 m.

🔺 SOH · CAH · TOA

6.2 · right-angled ratios
sin θ = opp ⁄ hyp cos θ = adj ⁄ hyp tan θ = opp ⁄ adj
  • Find a side: rearrange (e.g. opp = hyp·sin θ).
  • Find an angle: use sin⁻¹, cos⁻¹, tan⁻¹.

Label opp / adj relative to the angle you are using.

🧭 Elevation & bearings

6.2 · applications
  • Elevation = angle up from horizontal; depression = angle down. They are equal (alternate angles).
  • Bearings: 3 figures, clockwise from North (e.g. 072°, 240°).

Example: from 40 m away the elevation to a temple gopuram top is 35° → height = 40·tan 35° = 28.0 m.

6.5–6.6 Non-right-angled triangles & 3D Extended

📏 Sine rule

6.5 · a side + its opposite angle
a⁄sin A = b⁄sin B = c⁄sin C
  • Finding a side: keep sides on top.
  • Finding an angle: flip → sin A⁄a = sin B⁄b.

Side always sits opposite its angle. Ambiguous case: given 2 sides + a non-included angle, a second answer 180°−θ may also fit.

📐 Cosine rule

6.5 · no opposite pair

Find a side · (2 sides + included ∠)

a² = b² + c² − 2bc·cos A

Find an angle · (3 sides)

cos A = (b² + c² − a²) ⁄ 2bc

The angle A is always opposite the side a. Use when the sine rule can't (no side–opposite-angle pair).

🟩 Area & which rule

6.5 · area + choosing
Area = ½·a·b·sin C

Two sides and the included angle C between them.

Which rule?

  • Right angle → Pythagoras / SOHCAHTOA.
  • 3 sides, or 2 sides + angle between → cosine rule.
  • A side with its opposite angle → sine rule.

🧊 3D trigonometry

6.6 · Pythagoras & trig in 3D
  • Break the solid into right-angled 2D triangles, then use Pythagoras / SOHCAHTOA.
  • Space diagonal of a cuboid = √(l² + w² + h²).
  • Angle between a line and a plane: drop a perpendicular to the plane, form a right triangle, use tan/cos.

6.3–6.4 Graphs, exact values & solving equations Extended

y = sin x

6.4 · period 360°
0°90°180°270°360°

0 → max +1 at 90° → 0 → min −1 at 270° → 0. Zeros 0, 180, 360°.

y = cos x

6.4 · period 360°
0°90°180°270°360°

max +1 at 0° & 360° → 0 at 90° → min −1 at 180° → 0 at 270°. Zeros 90, 270°.

y = tan x

6.4 · period 180°
0°90°180°270°360°

Zeros 0, 180, 360°; asymptotes at 90° & 270° (undefined). Repeats every 180°, range all ℝ.

Hand trick

Under 0°·30°·45°·60°·90° write 0 1 2 3 4. Then sin = √n ⁄ 2; cos = the same list reversed; tan = sin ÷ cos.

🔑 The partner rules

solving · 0° ≤ x ≤ 360°

Calculator gives the first angle x₁. The second solution is:

sin x = k → 180° − x₁
cos x = k → 360° − x₁
tan x = k → 180° + x₁

Then +360° to any negative answer; keep only values in 0°–360°.

🧭 Method — every time

solving trig equations
  1. Rearrange to sin/cos/tan x = k.
  2. First angle from the positive value of k.
  3. Apply the matching partner rule.
  4. Adjust into range; check both fit.

No solution if |k| > 1 for sin or cos. tan always gives two answers.

✏️ Worked · cos

2 cos x + 1 = 0
cos x = −½ cos⁻¹(½) = 60° −ve → 180 − 60 & 180 + 60 x = 120°, 240°

cos is negative in the 2nd & 3rd quadrants → 120° and 240°.

✏️ Worked · sin

sin x = 0.6
x₁ = sin⁻¹0.6 = 36.9° 180 − 36.9 = 143.1° x = 36.9°, 143.1°

Positive sin → 1st & 2nd quadrants; partner is 180 − x₁.

✏️ Worked · tan

tan x = −1.5
from +1.5: tan⁻¹1.5 = 56.3° 180 − 56.3, 360 − 56.3 x = 123.7°, 303.7°

Negative tan → 2nd & 4th quadrants → 123.7° and 303.7°.

💡 Why two answers?

the graph view

"Solve sin x = 0.5" asks where the flat line y = 0.5 cuts the sine wave. It hits twice in 0°–360°: the calculator gives one crossing (30°), the graph's symmetry gives the other (150°).

📌 Memory hooks

lock these in
  • Sine → 180 − x
  • Cos → 360 − x
  • Tan → 180 + x

sin & cos reflect (stuck in −1..1); tan repeats every 180° and reaches every value.

🔁 Different range?

e.g. −180° to 180°

If the interval isn't 0°–360°, find all answers with the partner rule first, then shift each by ±360° until it lands inside the range asked. Keep every value that fits.
